# TGVerity v0.2 Design — 2026-07-04 20:40 MSK

## Goal

Promote `tgverity-core` from a framing prototype to a **testable protocol core**: crypto-ready packet format, ACK + state machine, full platform-adapter contract, storage interface, and leveled logging. Prepare real Telegram Desktop hooks. **Protocol-first:** plaintext body via an insecure crypto provider, with a crypto-ready format and seam.

## Scope

- **A (this cycle):** `tgverity-core` v0.2 lib + logging + desktop hook patches/shim + GUI build attempt.
- **Deferred:** X3DH key exchange + Double Ratchet messaging (README: needs external crypto review); libsodium AEAD provider (config-flag fast-follow, format-stable); session state machine (Signal session layer spec).

## Crypto stance

```text
CryptoProvider interface: seal(plaintext, aad)->bytes | open(ciphertext, aad)->opt<bytes> | suiteId()
Default = IdentityCryptoProvider (no-op, INSECURE). Logger emits INSECURE MODE on every seal/open.
Packet carries crypto_suite field; body = provider output.
Swap to libsodium = new provider + flag, ZERO format change.
```

## Packet format (v1, extended)

Envelope (safe-alphabet encoded transport, unchanged prefix):

```text
protocol=tgverity
version=1
crypto_suite=<id>
type=<relay_text|relay_ack>
packet_id=<counter>
ref_id=<id|->
body_safe=<provider output, safe-encoded>
```

- `packet_id`: monotonic counter per session (replaces fixed `"prototype-1"`).
- `type`: `relay_text` or `relay_ack` (ACK carries `ref_id` = original `packet_id`).
- Prefix unchanged: `TGVerity secure packet. Open with TGVerity to read.\nv1:`.

## State machine

```text
local_pending → tg_sent → tgverity_ack → cleanup_pending → cleanup_done
                       ↘──────────────────────────────────► failed
```

Failed cleanup never invalidates a stored message.

## Adapter contract (`tg_client`)

```text
sendPacketText(chatId, text) -> sendId
onMessageIdBound(sendId, serverId)        // callback
onIncomingText(chatId, text)              // callback
deleteMessagesRevoke(chatId, ids)         // single revoke call, ≤100 ids, never self-then-revoke
suppressNotification(serverId) / suppressEdit(serverId)
```

Implementations: `FakeTelegramClient` (in-memory, tests), `StubTelegramClient` (stdout), `TdlibTelegramClient` (shim, real wiring later).

## Logging

`log.h`: levels trace/debug/info/warn/error; component tag; stderr + optional `TGVERITY_LOG_FILE`; redact plaintext/keys (`[redacted]`); `INSECURE MODE` banner. Wired through core/adapter/bridge.

## Bridge orchestration

```text
Send : plaintext → provider.seal → packet(local_pending) → adapter.sendPacketText → bind → tg_sent
Recv : onIncomingText → parse → provider.open → store virtual msg → emit ACK
ACK  : recv ACK(ref_id) → mark tgverity_ack → schedule cleanup
Clean: deleteMessagesRevoke([msgId, ackId]) once, after ACK + ids known
```

## Desktop v0.2 hooks (patch + shim; GUI build separate)

Real hooks per `docs/message-lifecycle.md` hook map: send-wrap (`api_sending`), incoming (`api_updates` `updateNewMessage`), delete-revoke (`data_histories` `deleteMessages` `revoke=true`), notif suppress (`skipNotification`), edit suppress (`context_menu`). A `tgverity_bridge_shim` in the tdesktop tree mirrors the core format (C ABI later). v0.1 branding patch retained.
